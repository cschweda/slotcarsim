### Added

- ROADMAP.md — post-POC direction (more tracks, multiplayer, Tyco TCR mode).
- Project scaffold: Vite + TypeScript (strict) + vitest, exact-pinned dependencies, Netlify static deploy config.
- Fixed-timestep accumulator loop (`src/loop.ts`), with hidden-tab-safe reset, built test-first.
- Sim/config purity guard (`src/sim/purity.test.ts`) mechanically enforcing that `src/sim/` and `src/config/` stay pure TypeScript, with no Three.js, DOM, or nondeterministic calls.
- Initial tuning constants (`src/config/tuning.ts`) and skeletal simulation types (`src/sim/types.ts`).
- Photoreal look-dev scene (`src/render/scene.ts`, `src/render/lookdev.ts`): ACES filmic tone mapping, `RoomEnvironment` image-based lighting, a warm shadow-casting key light, and clearcoat/chrome test spheres.
- Project design document (`docs/2026-07-17-slotcar-sim-design.md`).
- Pure 2D vector/angle math helpers (`src/sim/math.ts`): `Vec2`, `add`/`sub`/`scale`/`rot`/`len`/`dist`, `wrapAngle` (into `(−π, π]`), `clamp`, `lerp` — built test-first.
- Authentic AFX track-piece catalog (`src/sim/track/pieces.ts`): straights (15/9/6/3 in) and 90°/45° curves (6/9/12 in radius), converted from inches to meters.
- Track builder (`src/sim/track/builder.ts`, TDD): `buildTrack()` walks a `PieceRef[]` list into a closed, 2-lane `Track`, deriving each lane's exact line/arc segment from the signed lane offset (`+d` lane 0, `−d` lane 1) and validating closure (throws a descriptive error naming the gap, heading error, and piece count on a bad layout).
- Arc-length-parametrized lane path (`src/sim/track/path.ts`, TDD): `createLanePath()` gives exact `pointAt(s) → {pos, heading, curvature}` via binary search over precomputed segment lengths, with modulo wrapping for any real `s` (including negative).
- Classic oval track definition (`src/config/tracks.ts`): 2 straights + a 180° left corner, twice — centerline lap ≈2.96 m, inner lane ≈2.84 m.
- Flat debug track view (`src/render/debugView.ts`): per-lane ribbon (cyan/orange), grey piece-joint markers, and two dot meshes positioned per frame.
- `src/main.ts` now builds and displays the oval by default (ground plane + debug view + two dots orbiting their lanes at constant speed); the M0 look-dev spheres move behind a `?lookdev` query param.
- Authentic AFX resistor-controller motor model (`src/sim/car/motor.ts`, TDD): `effectiveVolts()` (authentic resistor-divider curve / linear / stepped-quantized response modes), `driveAccel()`, `brakeAccel()` (pure `−brakeK·v`, no drag term, so brake distance is the exact closed form `(v0−v1)/brakeK`), and `carAccel()`'s deadband switch between them.
- `TUNING` (`src/config/tuning.ts`) extended with the motor's full constant set (`supplyV`, `motorR`, `controllerR`, `accelPerVolt`, `backEmfK`, `rollingDrag`, `throttleDeadband`, `responseMode`, `steppedBands`, `keyboardRampRate`) plus a shared `Tuning` type every sim/input/ui module reads through — one mutable object, read fresh every step, so the dev tuning panel can change feel live.
- Sub-tick lap timing (`src/sim/timing.ts`, TDD): `createLapTimer()` interpolates the exact fractional-tick crossing time when a car's step wraps past `s=0`, so lap times aren't quantized to the ~8ms tick and stay accurate to 1e-6s for a constant-speed car regardless of dt phase.
- Deterministic `Sim` world (`src/sim/world.ts`, TDD): `createSim()` steps every car with semi-implicit Euler (`carAccel` → `v` → `s`), keeps one-step-behind `prevCarStates()` alongside `carStates()` for render interpolation, supports `constant`-controlled cars (the pace-car placeholder) that ignore input and hold their velocity exactly, and feeds each car's lap timer. Bit-identical across repeated runs of the same scripted input trace (`src/sim/determinism.test.ts`) — the property multiplayer/ghost-replay will build on later.
- `wrapLerp(a, b, alpha, L)` (`src/sim/math.ts`, TDD): interpolates a car's arc-length position by the shortest **forward** hop modulo the lane length, so render interpolation stays smooth through a lap-line wrap instead of lerping backward across most of the lap.
- `CarState` gains `lapCount`; new `SimEvent` type (`{type:'lap', carIndex, lapNumber, lapTimeSec}`) (`src/sim/types.ts`).
- Input abstraction (`src/input/`): `ThrottleSource` interface; `createKeyboardThrottle()` (hold Space/ArrowUp to ramp at `keyboardRampRate`/s, release snaps to 0 immediately, prevents default scrolling); `createGamepadThrottle()` (re-fetches `navigator.getGamepads()` every read — never caches a `Gamepad` reference — reads `buttons[7]` on a `standard` mapping or falls back to scanning buttons/axes on a non-standard one, with a 0.03 deadzone); `createInputManager()` (prefers a connected gamepad, falls through to keyboard — including the authentic "dropped controller full-brake" on a mid-race disconnect).
- HUD (`src/ui/hud.ts`): fixed top-left overlay — lap count, last/best lap times (3 decimals, `—` before the first lap), a vertical throttle bar, and the active input source.
- Dev tuning panel (`src/ui/debugPanel.ts`, dev-only or `?tune`): live-bound range sliders for the motor constants, a `responseMode` select, numeric `v`/throttle readouts, and a strip chart of the last ~4s of both.
- `src/main.ts` now drives the player car (lane 0) from `inputManager`, circulates a constant-velocity pace car (lane 1, 1.5 m/s), renders both via `wrapLerp`-interpolated positions between sim ticks, and wires lap events into the HUD; `?lookdev` behavior is unchanged.
- Seeded PRNG (`src/sim/rng.ts`, TDD): `createRng(seed)` (mulberry32) → `{next(): number}` in `[0, 1)` plus a `range(lo, hi)` helper — the sim's only source of randomness, so deslot/tumble kinematics stay reproducible from a fixed seed.
- Lateral grip model (`src/sim/car/cornering.ts`, TDD): `stepCornering()` — first-order-filtered cornering demand (`aLat = v²·|κ|`, "chassis takes a set" via `latFilterTau`), progressive tail-out slide (`slideYaw` follows `sign(κ)` at `slideTau`, capped at 0.6 rad) with proportional speed scrub once past `gripSoft`, and a dwell-gated deslot trigger (`gripHard` sustained for `deslotDwell` seconds) so a single-tick curvature spike at a straight/curve joint can't spuriously deslot a car at legal speed.
- Tumble/marshal state machine (`src/sim/car/deslot.ts`, TDD): `beginTumble()` draws exactly once from the shared rng (kick fraction, spin rate) to set the initial tumble velocity/spin — deterministic from that point on — through fixed-duration `tumbling` → `waiting` phases (`tumbleFriction` decelerates the slide, spin optionally decays) and back onto the lane at the exit arc-length, `v=0`, with `generation` incremented. Total penalty is exactly `tumbleDuration + marshalDuration` = 2.0s. `tumblePose()` exposes `{phase, progress, pos, yaw}` for the renderer without advancing state.
- `src/sim/car/car.ts` (TDD via existing + new suites): `stepCar()` extracted from `world.ts` — phase dispatch composing motor/brake + cornering + integration in `'slot'`, handing off entirely to the deslot state machine (throttle ignored) in `'tumbling'`/`'waiting'`. `world.ts` is now orchestration only: it owns the one seeded rng shared by all cars (draw order = car order, deterministic) and feeds each car's lap timer; the M2 semi-implicit-Euler/constant-car/lap-timing behavior for the slot phase is unchanged.
- `CarState` gains `slideYaw`, `aLatFiltered`, `hardTicks`, `phase` (`'slot' | 'tumbling' | 'waiting'`), `phaseTicks`, `generation`, `tumble: TumbleState | null`; new `TumbleState` and `SimEvent` variants `{type:'deslot', carIndex, atS, speed}` / `{type:'reslot', carIndex}` (`src/sim/types.ts`). `createSim()` gains a `seed` option (default 1).
- `TUNING` (`src/config/tuning.ts`) extended with the M3 cornering/deslot constants: `gripSoft`, `gripHard`, `latFilterTau`, `slideTau`, `yawPerAccel`, `scrubPerAccel`, `deslotDwell`, `tumbleDuration`, `marshalDuration`, `tumbleFriction`.
- `determinism.test.ts` extended: a full-throttle-into-the-9″-curve trace that provably deslots and reslots, run twice for bit-identical final state and event lists; a different-seed run proves the deslot/reslot event *sequence* is seed-independent (the trigger is pure physics) while the tumble's rest kinematics differ (the only rng-dependent part).
- Debug rendering (`src/render/debugView.ts`): the two dot spheres are replaced by small flat boxes (30×12mm) via `setCarPoses(poses: {x, y, yaw, elevated?}[])`, so slide/tumble yaw is visible; `elevated` (tumbling/waiting) lifts the box ~10mm.
- `src/main.ts` computes each car's render pose from interpolated `s` → `pointAt` → heading + interpolated `slideYaw` while slotted, or from `tumblePose()` while tumbling/waiting; a GENERATION GUARD snaps straight to the current pose (no interpolation) the tick a reslot happens, so the box doesn't sweep backwards across the table from its tumble landing spot to the lane exit point.
- Dev tuning panel (`src/ui/debugPanel.ts`) gains sliders for `gripSoft` (4–30), `gripHard` (6–36), `yawPerAccel` (0–0.2), and `scrubPerAccel` (0–2).
- Photoreal track mesh (`src/render/trackMesh.ts`, TDD): `createTrackMesh(track)` extrudes the authentic AFX cross-section — near-black roadbed slab (`#1a1a1c`, chamfered outer edges, procedural roughness variation), 3mm×4mm recessed slots (`#0a0a0b`), four raised steel rails per section (`#b8b8bd`, metalness 1, biased env reflection so they read as bright streaks), lock-and-joiner seams (`#050506`) at every piece boundary, and segmented white snap-on guardrails (`#efece2`, clearcoat) on curve outsides — swept along the piece centerline (reconstructed as the exact midpoint of the two lane paths) with per-arc tessellation derived from a 0.2mm chord bound (`arcSegmentCount`). Merged by material via `BufferGeometryUtils.mergeGeometries` to exactly 5 draw calls for the whole static track. Tests: chord bound (incl. tightest 6″ radius), ≤5-mesh draw-call budget, non-empty NaN-free geometry, UV/normal presence, shadow flags, disposal.

- Car scene view (`src/render/carsView.ts`, TDD): `createCarsView(scene, track, styles)` replaces the box rendering — a pin-guided **chord** orientation (body points from a wheelbase-back rear reference to the pin, lagging the exact tangent) with slide-yaw added about the pin so the tail swings out, wheels spun by the distance actually travelled (`Δs/radius`, skipped across a reslot teleport), and a deterministic tumble: a double-bounce launch height plus pitch/roll that scale with the sim's `yawRate` and ease back to a flat landing, freezing flat while `waiting`. The `?debug` neon-box view is unchanged.
- Fully synthesized motor-voice math (`src/audio/mapping.ts`, TDD): `motorF0(v, vmax)` (`120 + 520·(v/vmax)` Hz, clamped to `[120, 640]`), `panForX(x, centerX, tableHalfWidth)` (signed offset from the table center, clamped to `[-1, 1]`), and `motorGains(throttle, v, vmax)` → `{tone, buzz, hiss}` — tone a 0.15-floor curve rising with v, buzz `throttle·(1 − v/vmax)` (peaks under load below top speed, the commutator "slipping brushes" noise), hiss `∝ v/vmax` (brush noise scales with rotation speed), jointly silent only when the car is stopped (`v < 0.01`) AND the trigger is released (`throttle < 0.02`) — otherwise buzz alone keeps a stalled-but-driven motor audible and tone/hiss alone keep a coasting car audible. 21 tests: f0 endpoints + clamp (incl. transient overspeed and defensive negative-v), pan clamp both sides + dead center, gains' silence condition, and monotonicity samples for all three curves.
- WebAudio engine (`src/audio/engine.ts`): `createAudioEngine()` → one `AudioContext` (constructed suspended, per the browser's autoplay policy) with a `master` `GainNode(0.7)` → `DynamicsCompressor` → destination bus; `ensureRunning()` marks the engine unlocked and resumes (call only from inside a real user-gesture handler); a `statechange` watchdog and a `visibilitychange` listener (`hidden` → `ctx.suspend()`, `visible` after unlock → resume) recover a context that drops to `suspended`/`interrupted` (tab hide, OS audio interruption) without ever resuming before the first real unlock.
- Persistent per-car motor voice (`src/audio/motorVoice.ts`): `createMotorVoice(engine, {detuneCents})` builds its whole node graph exactly once — triangle oscillator at f0 (tone) + square oscillator at 3·f0 through a tracking bandpass (Q 2, commutator buzz) + a once-generated, ctx-shared 2s looped white-noise buffer through a second tracking bandpass at 6·f0 (Q 0.8, brush hiss) — three branch gains into a shared voice gain → `StereoPannerNode` → `engine.master`. `update({v, throttle, x, vmax, tableHalfWidth, centerX})` moves every AudioParam via `setTargetAtTime` (τ≈0.03s) only — oscillator/filter frequencies from `motorF0`, branch gains from `motorGains`, pan from `panForX` — never creating a node after construction and never writing `.value` at audio rate. `detuneCents` (fixed at construction, 0 for the player / +26 (~+1.5%) for pace/AI) rides on the oscillators' own `.detune` param, so WebAudio's `frequency·2^(detune/1200)` combination gives the authentic two-motor beat with no extra per-frame math.
- One-shot sound effects (`src/audio/sfx.ts`): `createSfx(engine)` → `deslotClatter(pan)` (a 0.35s bandpass noise burst sweeping 1.8kHz→500Hz plus 4 descending square-wave clicks, panned to the car), `lapBeep()` (60ms quiet 1250Hz sine blip), `countdownBeep(final)` (880Hz/200ms, or 1320Hz/400ms when `final` — wired into the race start in M7). Every call builds a fresh transient graph (the correct pattern for one-shots, unlike the persistent voices) connected straight to `engine.master`, with gain/frequency envelopes scheduled once via `setValueAtTime`/`exponentialRampToValueAtTime` and self-release (stop + disconnect) on `'ended'`.
- Start gate (`src/ui/menus.ts`): `createStartGate(container, onStart)` — a full-screen "AFX SLOT CAR SIMULATOR / Click or press any key to start" overlay (vintage-simple, monospace, matches the HUD look) whose click/keydown handler consumes the event (`stopImmediatePropagation` + `preventDefault`) and calls `onStart()` synchronously, inside that same real user-gesture call stack — the one and only valid place to call `ctx.resume()` (a gamepad button press is not user activation in Chrome). M7 expands this file into real menus (track/car select, countdown).
- `src/main.ts` wiring: sim/render start immediately in attract mode (pace car circulating silently); the `AudioEngine`, one `MotorVoice` per car (player detune 0, others +26 cents), and the `Sfx` bank are created only inside the start gate's `onStart`. Every render frame feeds each car's voice its current `v`, world x (reused from the same interpolated pose already computed for `?debug`'s view, so panning doesn't need a second position calculation), and a throttle-for-gain-purposes value (`input`-controlled cars pass the real trigger reading, `constant`-controlled cars pass `v/vmax` so the pace car still sounds "driven"). `deslot` events call `deslotClatter(panForX(...))` (panned to the exact exit position, any car); the player's `lap` events call `lapBeep()`. `'M'` toggles master gain `0 ↔ 0.7` via `setTargetAtTime` (never a raw `.value=` jump, to avoid a click); the HUD shows a small "MUTED" indicator while active.
- HUD (`src/ui/hud.ts`) gains a `muted` field and a small red "MUTED" line, hidden unless active.
- Numeric audio verification (implementer, real Chrome tab, own port): a temporary AnalyserNode tapped on `master` (removed before commit) measured the tone branch's dominant peak against `motorF0(v)` at three distinct player speeds — 0.586 m/s → 221.6 Hz expected / 218.3 Hz measured (1.5% error), 1.216 m/s → 330.8 Hz / 328.1 Hz (0.8%), 2.041 m/s → 473.7 Hz / 470.2 Hz (0.7%, 0.1% with a longer settle) — all comfortably inside ±8%. The pace car's isolated peak (player silent) measured 385.25 Hz against a detuned expectation of 385.75 Hz (undetuned would be 380 Hz) — confirming the +26-cent detune — and remained a second, simultaneously-present peak distinct from the player's own moving peak at every sampled speed.
