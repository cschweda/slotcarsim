## [Unreleased]

### Added

- Persistent sound toggle (`src/ui/overlays.ts`'s `createSoundToggle`): a top-right `SOUND: ON`/`SOUND: OFF` button, `aria-pressed` + `aria-label`, visible in every screen after the start gate (menu, countdown, racing, results) via a z-index above the menu system's full-screen dialogs. Clicking it and pressing `M` both funnel through the same `main.ts` toggle path, so the two can never disagree about the current state.
- Sound preference persistence (`src/ui/soundPref.ts`): `loadSoundPref()`/`saveSoundPref()` read/write `localStorage` key `afx-sound` (`'on'`/`'off'`); any missing, corrupt, or unrecognized stored value reads back as `false`.
- Visible keyboard controls listing: the start gate's old single hint line is replaced with a two-column block (throttle, sound, menu/abort, menu navigation, gamepad) for players without a gamepad; the race setup menu's footer gains a matching one-line reminder (`Space/↑ = throttle · M = sound · Esc = abort`).
- Mouse-wheel/trackpad-pinch camera zoom (`src/render/cameraZoom.ts`): scroll to zoom the track view in/out (0.35×–1.15× the fitted framing — closer on the figure-8, where cars used to read small), smoothed via a frame-rate-independent exponential ease. Wheel input is captured on the canvas host only, so scrolling the docked tuning panel's own content is unaffected; zoom resets to the fitted default on every race/track rebuild.

### Changed

- Sound now defaults to **off** (was on): `main.ts`'s `muted` flag initializes from the persisted preference (`!loadSoundPref()`) instead of always starting unmuted. The start gate shows a truthful one-line summary of the current state ("Sound is off — press M or use the Sound button to enable" / "Sound on").
- HUD (`src/ui/hud.ts`): the small "MUTED" badge is removed — the persistent sound toggle button now carries that state, so the HUD no longer duplicates it.
- Dev tuning panel (`src/ui/debugPanel.ts`): docks as a collapsible right-hand column next to the track view instead of overlaying it (`?tune`/dev only) — you can race with it open. A header close button collapses it (the canvas reflows to fill the freed width); a small `TUNE` tab reopens it.
- AI driver personality retune (`src/sim/ai/driver.ts`) — "less than perfect, more human": throttle tremor amplitude roughly doubled (±3%/±8% → ±5%/±12% at d=1.0/0.35); brake-point misjudgment events are more frequent at every difficulty (every 12–25s / 7–16s, was 25–45s / 8–20s) and now two-sided — ~60% are a new UNDERSHOOT (braking early / lifting off, always safe regardless of difficulty) and ~40% are the original fallibility-scaled OVERSHOOT (unchanged — this is what keeps the d=1.0 zero-deslot guarantee); the d=1.0 reaction-delay floor moved from 40ms to 70ms. Determinism is untouched (same rng streams; a fixed seed still races bit-identically) — only the AI's own throttle decisions change. `headless.test.ts`'s pins moved with it (deliberately, not a regression): the d=0.35 pinned deslot example moved from worldSeed 3 (2 deslots) to worldSeed 1 (1 deslot); a new "not a robot" suite pins a per-seed settled-lap-time spread of ≥0.15s at every difficulty (0.35/0.65/0.9/1.0), proving the retune produces real lap-to-lap variance, not just numbers that happen to satisfy the deslot pins.

## [0.1.0] - 2026-07-18

### Added

- Auto quality ladder (`src/render/scene.ts`): a rolling 120-frame frame-time monitor steps the renderer DOWN through DPR (2 → 1.5 → 1.25), then key-light shadow map size (→ 1024), then shadows off entirely under sustained load (a full window averaging > 20ms), and back UP only after 10 straight seconds of a sub-12ms window average — never above the user's own `?quality` choice, which seeds both the starting tier and the ceiling. `buildLadderTiers`/`createLadderPolicy` are plain data plus an injected `applyTier` callback (no three.js/DOM dependency), so the tier-stepping decision is unit-tested (18 tests) without a real `WebGLRenderer`; `createQualityLadder` wires it to a live `SceneHandle`, forcing the key light's shadow render target to regenerate at its new size (three.js won't resize an existing one lazily). `carsView.ts` gains a cheap blob-shadow fallback for the shadows-off tier: a shared dark radial-gradient quad geometry/material with one `Mesh` per car, tracking its ground position every frame (staying grounded through a tumble) via `setBlobShadows()`. The dev debug panel now shows the rolling avg frame time and current tier label.
- Gamepad calibration wizard (`src/input/gamepad.ts`): on a non-`standard`-mapping pad's first-seen `gamepad.id`, a 5s window samples every button/axis's range of motion (standard-mapping pads skip this — `buttons[7]` is already trusted — unless `?calibrate` forces a re-run); the control with the largest observed range is picked, its rest/max persisted to `localStorage` keyed by `gamepad.id`. Reads while the wizard is sampling return 0; subsequent reads rescale rest..max → 0..1 through the same existing deadzone. If nothing moves during the window, no calibration is stored and reads fall back to the pre-M8 `discoverFallback` heuristic unchanged. `everConnected`/`calibrating`/`calibrationSecondsLeft` drive a HUD "squeeze the trigger to connect a gamepad" hint and a full-screen calibration overlay ("SQUEEZE AND RELEASE THE TRIGGER"). `inputManager.ts` gains `pollGamepad()` so connection detection/calibration can progress at the menu, not just mid-race, without ever touching the keyboard fallback's stateful ramp.
- Rumble (`src/input/gamepad.ts`): `rumbleOnDeslot()` (300ms, strong 0.9/weak 0.5) and `rumbleOnReslot()` (80ms light pulse) via `vibrationActuator.playEffect`, feature-detected and fire-and-forget — a rejected or synchronously-throwing `playEffect` is swallowed, never surfacing as an unhandled rejection. Wired to the player's own deslot/reslot sim events only, independent of the audio-unlock gate.
- Per-track camera framing (`src/main.ts`): `reframeCamera` now fits each track's own bounding box, scaling the tuned oval offset's x/z components independently against the oval's own half-extents instead of one blended "radius" ratio — fixes the figure-eight's lower lobe clipping off-frame (its bbox is a near-square shape with much more depth than the x-dominant oval) while reproducing the oval's exact original framing (both axis ratios are 1). Height scales by the larger of the two ratios. Audio pan center/half-width are now read from the current session's own track bbox instead of a hardcoded oval-shaped constant (0.85 half-width / 0.381 center), previously silently wrong for the figure-eight.
- Vintage-scoreboard HUD polish (`src/ui/hud.ts`): a visually prominent amber (`#ffb454`) lap counter, a race position badge (P1/P2, ranked by lap count then in-lap arc-length progress), a best-lap time that flashes on improvement, and a small bordered "MUTED" badge (was a full-width red line). The countdown overlay (`src/main.ts`) is now warm-white with a soft text-shadow glow, "GO" in green, and fixes a bug where GO was visible for only about one frame (~16ms) before the race-phase transition cleared it — it now holds for a fixed 700ms via its own timer, independent of how fast the race machine moves on to 'racing'.
- Pinned headless figure-eight AI race test (`src/sim/ai/figure8Headless.test.ts`, TDD): AI (d=0.65, lane 1) against an idle player, exactly 5 laps within a sim-time band, a pinned zero-deslot count for seed 1, and finite car state throughout — commits the M7 controller's transient browser probe as a permanent regression test.
- A face-normal smoke test for the hand-wound criss-cross crossing square (`src/render/trackMesh.test.ts`): asserts every computed vertex normal points up, guarding `buildCrossingSquare`'s hand-authored winding (verified to actually catch a flipped-winding regression, not vacuous).
- Menu accessibility (`src/ui/menus.ts`): both option lists (race setup, results actions) now use proper listbox/option ARIA (`role="listbox"` + `aria-activedescendant` on the container, `role="option"` + `aria-selected` on each row), and Tab is trapped on the dialog root while a menu (already `aria-modal`) is open.

- AI cornering speed profile (`src/sim/ai/speedProfile.ts`, TDD): `buildSpeedProfile(lane, cfg, difficulty)` samples a corner-limited speed cap every 10 mm (`vCap = sqrt(gripHard·margin(d)/|κ|)`, `margin(d) = 0.72 + 0.21·d`), then runs a cyclic backward braking pass that exploits the motor's exact closed-form braking law (`dv/ds = −brakeK`) to turn the caps into a brakeable target-speed envelope (full-out on straights, braking into corners). Difficulty-scaled human fallibility, all from the driver's OWN seeded rng: a reaction-delay ring buffer (40–150 ms), low-pass throttle tremor (±3 % at d=1 … ±8 % at d=0.35), and scheduled brake-point-misjudgment events (every 8–20 s at d≤0.5, 25–45 s above) whose 1.12–1.18 overshoot is fallibility-scaled so a d=1 line stays inside its corner headroom while a d=0.35 one occasionally deslots.

- `src/main.ts` renders the two AFX cars (player Porsche 917, pace Ferrari 512) via `createCarsView`; `computeCarRenderPose()` hands the view the sub-tick-interpolated lane `s` + `slideYaw` while slotted (the same M3 GENERATION GUARD snaps a reslot instead of interpolating) or the `tumblePose()` plan-view pose plus `yawRate` while tumbling. The `?debug` neon-box view is unchanged.
- Fully synthesized motor-voice math (`src/audio/mapping.ts`, TDD): `motorF0(v, vmax)` (`120 + 520·(v/vmax)` Hz, clamped to `[120, 640]`), `panForX(x, centerX, tableHalfWidth)` (signed offset from the table center, clamped to `[-1, 1]`), and `motorGains(throttle, v, vmax)` → `{tone, buzz, hiss}` — tone a 0.15-floor curve rising with v, buzz `throttle·(1 − v/vmax)` (peaks under load below top speed, the commutator "slipping brushes" noise), hiss `∝ v/vmax` (brush noise scales with rotation speed), jointly silent only when the car is stopped (`v < 0.01`) AND the trigger is released (`throttle < 0.02`) — otherwise buzz alone keeps a stalled-but-driven motor audible and tone/hiss alone keep a coasting car audible. 21 tests: f0 endpoints + clamp (incl. transient overspeed and defensive negative-v), pan clamp both sides + dead center, gains' silence condition, and monotonicity samples for all three curves.
- WebAudio engine (`src/audio/engine.ts`): `createAudioEngine()` → one `AudioContext` (constructed suspended, per the browser's autoplay policy) with a `master` `GainNode(0.7)` → `DynamicsCompressor` → destination bus; `ensureRunning()` marks the engine unlocked and resumes (call only from inside a real user-gesture handler); a `statechange` watchdog and a `visibilitychange` listener (`hidden` → `ctx.suspend()`, `visible` after unlock → resume) recover a context that drops to `suspended`/`interrupted` (tab hide, OS audio interruption) without ever resuming before the first real unlock.
- Persistent per-car motor voice (`src/audio/motorVoice.ts`): `createMotorVoice(engine, {detuneCents})` builds its whole node graph exactly once — triangle oscillator at f0 (tone) + square oscillator at 3·f0 through a tracking bandpass (Q 2, commutator buzz) + a once-generated, ctx-shared 2s looped white-noise buffer through a second tracking bandpass at 6·f0 (Q 0.8, brush hiss) — three branch gains into a shared voice gain → `StereoPannerNode` → `engine.master`. `update({v, throttle, x, vmax, tableHalfWidth, centerX})` moves every AudioParam via `setTargetAtTime` (τ≈0.03s) only — oscillator/filter frequencies from `motorF0`, branch gains from `motorGains`, pan from `panForX` — never creating a node after construction and never writing `.value` at audio rate. `detuneCents` (fixed at construction, 0 for the player / +26 (~+1.5%) for pace/AI) rides on the oscillators' own `.detune` param, so WebAudio's `frequency·2^(detune/1200)` combination gives the authentic two-motor beat with no extra per-frame math.
- One-shot sound effects (`src/audio/sfx.ts`): `createSfx(engine)` → `deslotClatter(pan)` (a 0.35s bandpass noise burst sweeping 1.8kHz→500Hz plus 4 descending square-wave clicks, panned to the car), `lapBeep()` (60ms quiet 1250Hz sine blip), `countdownBeep(final)` (880Hz/200ms, or 1320Hz/400ms when `final` — wired into the race start in M7). Every call builds a fresh transient graph (the correct pattern for one-shots, unlike the persistent voices) connected straight to `engine.master`, with gain/frequency envelopes scheduled once via `setValueAtTime`/`exponentialRampToValueAtTime` and self-release (stop + disconnect) on `'ended'`.
- Start gate (`src/ui/menus.ts`): `createStartGate(container, onStart)` — a full-screen "AFX SLOT CAR SIMULATOR / Click or press any key to start" overlay (vintage-simple, monospace, matches the HUD look) whose click/keydown handler consumes the event (`stopImmediatePropagation` + `preventDefault`) and calls `onStart()` synchronously, inside that same real user-gesture call stack — the one and only valid place to call `ctx.resume()` (a gamepad button press is not user activation in Chrome). M7 expands this file into real menus (track/car select, countdown).
- `src/main.ts` wiring: sim/render start immediately in attract mode (pace car circulating silently); the `AudioEngine`, one `MotorVoice` per car (player detune 0, others +26 cents), and the `Sfx` bank are created only inside the start gate's `onStart`. Every render frame feeds each car's voice its current `v`, world x (reused from the same interpolated pose already computed for `?debug`'s view, so panning doesn't need a second position calculation), and a throttle-for-gain-purposes value (`input`-controlled cars pass the real trigger reading, `constant`-controlled cars pass `v/vmax` so the pace car still sounds "driven"). `deslot` events call `deslotClatter(panForX(...))` (panned to the exact exit position, any car); the player's `lap` events call `lapBeep()`. `'M'` toggles master gain `0 ↔ 0.7` via `setTargetAtTime` (never a raw `.value=` jump, to avoid a click); the HUD shows a small "MUTED" indicator while active.
- HUD (`src/ui/hud.ts`) gains a `muted` field and a small red "MUTED" line, hidden unless active.
- Numeric audio verification (implementer, real Chrome tab, own port): a temporary AnalyserNode tapped on `master` (removed before commit) measured the tone branch's dominant peak against `motorF0(v)` at three distinct player speeds — 0.586 m/s → 221.6 Hz expected / 218.3 Hz measured (1.5% error), 1.216 m/s → 330.8 Hz / 328.1 Hz (0.8%), 2.041 m/s → 473.7 Hz / 470.2 Hz (0.7%, 0.1% with a longer settle) — all comfortably inside ±8%. The pace car's isolated peak (player silent) measured 385.25 Hz against a detuned expectation of 385.75 Hz (undetuned would be 380 Hz) — confirming the +26-cent detune — and remained a second, simultaneously-present peak distinct from the player's own moving peak at every sampled speed.
